Client SDK for communicating with local storage or cloud server

Usage for apps requiring local storage

  • Set local storage path for user data and commit log in your app
  • Sample code snippet to scan from local storage 
      var preference = AtClientPreference()
                   ..hiveStoragePath= <storage_path>
                   ..commitLogPath = <commit_log_path>
                   ..syncStrategy=SyncStrategy.IMMEDIATE
                   ..isLocalStoreRequired=true;         
  await AtClientImpl.createClient('@alice', 'persona', preference);
  var atClient = await AtClientImpl.getClient('@alice');
  var result = await atClient.getKeys();
  • For more samples refer to api/at_client/AtClient-class.html.

Usage for apps communicating directly to user's cloud server

  • Use this approach if you are familiar with @ protocol commands.
  • Set local storage to false
  • Sample code to lookup public information 
     var preference = AtClientPreference()
                  ..isLocalStoreRequired=false;         
 await AtClientImpl.createClient('@alice', null, preference);
 var atClient = await AtClientImpl.getClient('@alice');
 var result = await atClient
         .getRemoteSecondary()
         .executeCommand('plookup:publickey@bob\n', auth: true);
